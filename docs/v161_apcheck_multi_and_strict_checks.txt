V161: fixes multi-AP-Check sends and false extra checks. The PCK now merges checks_to_send.json writes and preserves ap_check_next_count so multiple AP Check symbols destroyed in one spin send multiple AP Check locations. The client expands ap_check_next_count to sequential AP Check locations. Payment checks are no longer inferred from run logs or lbal_run_state.json; they only come from the PCK successful-rent-payment hook. First-get Send: checks now require an actual added/got/received/destroyed log action to reduce false sends such as Key.
