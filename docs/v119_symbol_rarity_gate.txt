V119: Fixes metadata symbols leaking into wrong rarities. V118 loaded all vanilla symbols as metadata for default descriptions/icons, but received fallback/AP-allowed symbols could remain in whatever rarity bucket LBAL had rolled. The live filter now requires non-AP-Check symbols to match their native vanilla rarity before they can stay in an Add Symbol bucket, and received-fallback symbols are injected only into their native rarity bucket. This prevents Very Rare symbols such as Diamond from appearing at Payment 2 while keeping default description icons loaded.
